# Review Checklist

每次 agent 完成实现或修改后，必须按以下清单自检。

## 架构检查

- [ ] 新增文件是否放在正确目录？
- [ ] 是否违反单向依赖？
- [ ] 是否把业务逻辑写入 `drivers/`？
- [ ] 是否把硬件操作写入 `app/`？
- [ ] 是否让 `protocols/` 直接操作硬件？

## 编码检查

- [ ] public API 是否清晰？
- [ ] 是否有未受控全局变量？
- [ ] 是否存在 magic number？
- [ ] 是否存在无边界 buffer 写入？
- [ ] 是否存在不合理动态内存？

## 运行时检查

- [ ] 是否存在阻塞等待？
- [ ] 是否存在 ISR 复杂逻辑？
- [ ] 是否存在未保护共享状态？
- [ ] 是否正确处理 DMA buffer 所有权？

## 质量检查

- [ ] 是否有错误处理路径？
- [ ] 是否有日志或诊断策略？
- [ ] 是否说明测试或验证方式？
- [ ] 是否存在 `NEED_CONFIRMATION`？

## 输出检查

- [ ] 是否列出修改文件？
- [ ] 是否说明所属层级？
- [ ] 是否说明 public API 变化？
- [ ] 是否说明未完成项或人工确认项？
