# 错误处理、日志和诊断规范

## 错误码

- MUST: 所有可失败模块必须返回明确错误码或状态。
- MUST: 错误码必须集中定义在 `common/error_code.h` 或等价文件中。
- MUST: 错误码必须区分参数错误、超时错误、硬件错误、协议错误、资源不足和状态非法。
- MUST NOT: 不允许静默吞掉错误。

## 错误处理路径

- MUST: 通信超时、CRC 错误、帧长度错误、参数非法、硬件初始化失败必须有明确处理路径。
- MUST: 关键错误必须上报到 `fault_manager`、`diagnostics_service` 或等价模块。
- MUST: 可恢复错误必须说明重试、降级或状态恢复策略。
- MUST NOT: 不允许只打印日志而不返回或记录错误状态。

## 日志

- MUST: 日志接口必须统一封装。
- MUST: 日志必须支持编译期开关和等级控制。
- MUST NOT: 不允许在业务代码中散落裸 `printf()`。
- MUST NOT: 不允许在 ISR 中直接打印日志。
- SHOULD: 关键模块应记录最近错误码、错误次数和关键状态。

## 诊断

- MUST: 关键模块应提供最小诊断接口。
- SHOULD: 通信模块应提供发送次数、接收次数、超时次数、CRC 错误次数等统计信息。
- SHOULD: 存储模块应提供写入失败次数、校验失败次数等统计信息。
- SHOULD: 诊断数据应避免暴露模块 private 状态的可写指针。
