# 调度和状态机规范

## 状态机

- MUST: 复杂流程必须优先使用显式状态机。
- MUST: 状态机状态必须使用 enum 或项目统一状态类型。
- MUST: 状态转换必须有明确触发条件。
- MUST: 超时必须基于系统 tick 或 RTOS tick。
- MUST: 状态机必须定义错误状态、恢复路径或失败返回。
- MUST NOT: 不允许通过长 delay 模拟状态等待。

## 调度

- MUST: 周期性调度必须可预测。
- MUST: 每个周期任务必须说明周期、最大执行时间预期和超时策略。
- MUST: 高优先级任务不得执行低价值长耗时工作。
- SHOULD: 通信、采样、控制、日志、诊断应按实时性分配优先级。

## 非阻塞流程

- MUST: 长流程必须拆分为多个可恢复步骤。
- MUST: 每一步必须能够快速返回或等待事件再次进入。
- MUST NOT: 不允许在状态机内部执行无超时阻塞等待。
- SHOULD: 对协议请求、Flash 写入、传感器转换等异步动作使用 `start + poll/complete` 模式。
